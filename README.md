# Illustrator Artboard Scripts

웹배너 작업할 때 반복적으로 겪는 대지(아트보드) 관련 문제들을 해결하기 위해 만든 일러스트레이터 ExtendScript(.jsx) 모음이다.

## 왜 만들었나

여러 사이즈의 배너를 한 파일 안에서 대지별로 작업하다 보면 아래 두 가지 문제를 자주 겪는다.

1. **PNG로 내보내면 원본보다 1px씩 더 크게 저장되는 문제** — 대지 좌표가 정수가 아니면(소수점 pt 값) 내보내기 과정에서 반올림 오차로 여분의 픽셀이 붙는다. 배너 광고 등록 시스템처럼 사이즈가 정확히 안 맞으면 오류가 나는 곳에 올릴 때 특히 골치 아프다.
2. **이미지를 여러 개 배치하고 나서, 그 크기에 맞는 대지를 하나하나 손으로 만드는 게 비효율적인 문제** — 이미지 개수가 많아지면 사이즈 재고 대지 만들고 이름 붙이는 작업만 한참 걸린다.

## 포함된 스크립트

### `대지좌표_정수보정.jsx`

문서 안 모든 대지의 좌표를 정수로 보정한다. 대지만 옮기면 안에 있는 그림과 어긋나 보이므로, 대지가 이동한 만큼 그 안의 오브젝트도 같이 이동시켜 상대 위치를 유지한다.

일러스트레이터는 대지 한 변이 227인치를 넘으면 자동으로 "큰 캔버스" 모드로 전환되면서 내부적으로 배율(Scale Factor)이 걸린다. 이 경우 화면에 보이는 숫자와 실제 좌표값 사이에 배수 차이가 생기는데, 이걸 모르고 반올림하면 엉뚱한 크기로 튀어버린다. 이 스크립트는 문서의 배율을 자동 감지해서, 화면에 보이는 값 기준으로 정확히 반올림하도록 처리했다.

(자세한과정: https://soybeanfield.tistory.com/entry/대지좌표수정)

**사용법**
1. 일러스트레이터에서 대상 파일을 연다
2. `File > Scripts > Other Script...` 로 이 파일을 불러온다
3. 완료되면 몇 개 대지를 보정했고, 감지된 배율이 무엇인지 알림창으로 알려준다

### `이미지별_대지생성.jsx`

문서 안에 배치(링크)되었거나 임베드된 이미지들을 찾아서, 각 이미지 크기에 맞는 대지를 자동으로 만들고, 대지 이름을 이미지의 원본 파일명(확장자 제외)으로 설정한다.

- 오브젝트를 선택한 상태로 실행하면 선택한 이미지만 처리하고, 아무것도 선택하지 않으면 문서 전체의 이미지를 대상으로 처리한다.
- 파일명에 한글이 포함되어 퍼센트 인코딩(`%EC%95%88...`)된 상태로 읽히는 경우 자동으로 디코딩한다.
- 이름이 중복되면 `_2`, `_3`처럼 자동으로 구분한다.

**사용법**
1. 이미지들이 배치되어 있는 문서를 연다
2. (선택 사항) 특정 이미지만 처리하고 싶으면 먼저 선택
3. `File > Scripts > Other Script...` 로 이 파일을 불러온다

## 설치 (매번 찾기 귀찮다면)

아래 경로에 `.jsx` 파일을 복사해두면 `File > Scripts` 메뉴에 항상 뜬다.

```
Applications/Adobe Illustrator [버전]/Presets.localized/[언어 폴더]/Scripts/
```

복사한 뒤 일러스트레이터를 재시작하면 메뉴에 반영된다.

## 주의사항

- `이미지별_대지생성.jsx`는 화면에 보이는 크기(geometricBounds) 그대로 대지를 만든다. 이미지가 확대/축소되어 배치되어 있으면 대지도 그 배율 그대로 만들어지니, 원본 픽셀 크기로 맞추고 싶다면 이미지를 100% 크기로 되돌린 뒤 실행하는 걸 권장한다.
- 두 스크립트 모두 실행 전 원본 파일을 백업해두는 걸 추천한다.
